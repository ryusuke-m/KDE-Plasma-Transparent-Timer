## アプリケーションの主な機能

### 1. 美しいガラスモーフィズムとドラッグ移動
* ウィンドウ背景には半透明のブラー（ぼかし）と極細のボーダーをあしらい、どんなデスクトップ壁紙にもなじむプレミアムな質感を持たせています。
* ウィンドウ上部（ドラッグゾーン）をドラッグして、画面上の任意の場所に自由に配置できます。
* **文字・UI要素の自動スケーリング**: ウィンドウサイズを変更すると、タイマーの時間表示やテキスト、各種要素のフォントサイズがウィンドウの縦横比率に合わせて自動的に拡大縮小します。

### 2. ストップウォッチ ＆ カウントダウン
* **ストップウォッチ**: 経過時間を累積します。
* **カウントダウン**: 設定時間（1〜720分）に達すると、デスクトップ通知と警告音でゲームのしすぎや作業時間を知らせます。

### 3. アプリケーション別の自動滞在時間記録（KDE Wayland対応）
* バックグラウンドでアクティブなウィンドウを自動トラッキングし、ウィンドウを切り替えるとリアルタイムで計測対象アプリ名（例: `google-chrome`, `kate` など）がタイマー下部に表示されます。
* 計測結果は、本日の「使用時間ログ」画面に美しい円グラフ（Chart.js）と割合（%）の一覧リストで自動表示されます。ウィンドウを大きく引き伸ばすと、グラフやログの表示領域もウィンドウ全高に合わせてダイナミックに拡大されるため、多くの履歴を快適に一覧できます。

### 4. マウスクリック透過（Click-Through）モード
* 設定画面のトグルスイッチ、またはグローバルショートカットキー **`Alt+T`** でクリック透過モードの切り替えが可能です。
* 透過モードが有効な間は、タイマーが表示された状態のまま、その下にあるエディタやブラウザ、ゲームを直接クリックして操作できます。
* 透過モードを解除したい場合は、もう一度 **`Alt+T`** を押すだけで通常操作に戻せます。

### 5. 将来の可視化に向けた拡張データ設計
* 記録データは `time_records.json` に時系列セッション（`timestamp`, `app`, `title`, `duration`）として記録されます。これにより、将来的に「日次・週次グラフ」や「曜日ごとのゲーム時間比較」などの機能を簡単に追加できます。

## 動作確認手順

アプリは現在、バックグラウンドタスクとしてデスクトップ上で実際に起動しています。

1. **基本操作**:
   - タイマーの上にマウスを乗せると、各種ボタン（再生、リセット、グラフ、設定）が表示されます。
   - 再生ボタンで計測を開始します。
2. **自動記録の確認**:
   - ブラウザや他のアプリケーションにウィンドウを切り替えてから数秒間操作し、タイマー下部のアプリ名表示が切り替わるか確認します。
   - グラフアイコン（棒グラフのボタン）を押して、アプリごとの計測結果が正しくドーナツグラフに反映されているか確認します。
3. **クリック透過の確認**:
   - 設定アイコンから設定画面を開き、「マウスクリック透過」をオンにします。
   - タイマーの上がクリックできなくなり、後ろのエディタなどが触れることを確認します。
   - キーボードの **`Alt+T`** を入力し、透過モードが解除されて再度タイマーを操作できる状態に戻ることを確認します。

## アプリの起動・終了方法

### 1. 起動方法
ターミナルを開き、以下のコマンドを実行します。
```bash
cd /home/administrator/Documents/TranspalentTimer
npm start
```
※Linux/KDE特有のセキュリティおよびサンドボックス制限を自動で回避（`--no-sandbox`）して起動します。

### 2. 終了方法
* **UIから終了する場合**:
  タイマーにマウスカーソルを合わせると、右上に小さな **「×（閉じる）」** ボタンが表示されます。これをクリックすると安全にアプリが終了し、KWinスクリプトのクリーンアップと現在の時間記録の保存が自動で行われます。
* **ターミナルから終了する場合**:
  `npm start` を実行したターミナル上で **`Ctrl + C`** を入力してプロセスを終了します。
* **強制終了する場合**:
  バックグラウンドで起動してしまい画面上から消せなくなった場合は、以下のコマンドでプロセスを終了できます。
  ```bash
  pkill -f "electron"
  ```
