# 透明なデスクトップタイマーアプリ（作業・ゲーム時間計測＆制限対策）の実装計画

画面上に透明に表示され、作業時間やゲーム時間の測定・制限ができるデスクトップタイマーアプリを作成します。
KDE Wayland環境に対応した「使用中アプリケーションの自動時間記録」機能も搭載します。

## ユーザーレビューが必要な項目

> [!IMPORTANT]
> **アプリケーションの自動時間記録について（KDE Wayland対応）**
> - 本アプリは、KDEのウィンドウマネージャー（KWin）のスクリプト機能を利用して、現在アクティブな（最前面にある）アプリケーションのプロセス名とウィンドウタイトルを自動検出します。
> - この検出のため、アプリの起動時に一時的にKWinスクリプトを登録し、システムのユーザーログ（`journalctl`）から切り替えイベントを検知する仕組みを実装します。動作中に一時的な管理者権限等は不要ですが、システムログへのアクセスを必要とします。

> [!TIP]
> **デザインと使いやすさへのこだわり**
> - デスクトップの壁紙や他のウィンドウに調和する、美しい**ガラスモーフィズム（半透明）デザイン**を採用します。
> - マウスをホバーしたときだけ操作ボタン（一時停止、設定、グラフ確認など）がふわっと浮き出るようにし、作業やゲームの邪魔にならないシンプルな表示にします。
> - **クリック透過モード**をサポートします。タイマーを表示したまま、その下にあるウィンドウを直接クリックして操作できるようになります（ショートカットキーまたは切り替えボタンで切り替え）。

### 2. バックエンド（メインプロセス）

#### [NEW] [main.js](file:///home/administrator/Documents/TranspalentTimer/main.js)
- Electronメインプロセス。
- 透明で常に最前面に表示されるフレームレスウィンドウを作成。

- `journalctl` ログをストリーミング監視してアクティブアプリの切り替えを検知。
- アプリごとの使用時間を記録するデータのファイル保存（`time_records.json`）。将来的な詳細可視化（カレンダー表示、週次レポート、時間帯ごとの分析など）を考慮し、単純な累積時間だけでなく、「いつ（タイムスタンプ）」「どのアプリが（プロセス名/タイトル）」「どれだけ（秒数）」アクティブだったかを時系列のセッションログ形式で記録できる拡張性の高いデータ構造で設計します。

#### [NEW] [kwin_script.js](file:///home/administrator/Documents/TranspalentTimer/kwin_script.js)
- KWin用のJavaScriptスクリプト。アクティブウィンドウの切り替えイベントを検知し、アプリケーション情報をシステムログに出力します。

- プレミアムなタイポグラフィのために Google Fonts から Inter フォントをインポートします。

#### [NEW] [renderer.js](file:///home/administrator/Documents/TranspalentTimer/renderer.js)
- フロントエンドのロジック。
- カウントダウンタイマーおよびストップウォッチの管理。

- 設定（制限時間の設定、クリック透過の切り替えなど）のUI制御。

## 検証計画 (Verification Plan)

### 自動テスト（動作検証）
- `npm run start` でアプリケーションが正常に起動することを確認。

### 手動検証
1. **透明表示と最前面表示 of ウィンドウ**:
   - ウィンドウが透明（背景透過かつぼかしあり）で、他のアプリより常に最前面に表示されるか。
   - ドラッグして画面上の任意の場所に移動できるか。
2. **クリック透過の確認**:
   - クリック透過モードをONにしたとき、タイマーをクリックしてもその下にあるブラウザやエディタを操作できるか。
3. **自動記録機能の確認**:
   - ブラウザ（Chromeなど）、エディタ（VS Code / Antigravityなど）、ゲームなどのウィンドウを切り替えたとき、アプリごとの計測時間が正しくインクリメントされるか。
   - アプリを切り替えて数分放置したのち、履歴画面で計測結果がChart.jsによってグラフ表示されるか。
4. **タイマー機能の確認**:
   - カウントダウンタイマーが0になった際に通知（デスクトップ通知または警告エフェクト）が発生するか。
